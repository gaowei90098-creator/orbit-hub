## 0.5.3 — Codex CLI StdIO 权限修复（2026-06-16，`v0.5.3`）

**主题**：修复 Codex CLI 在 AgentHub 中自述只读、无法执行 PowerShell/定位本机项目的问题。

- Codex CLI 0.134 的 `codex exec --sandbox workspace-write` 在非交互派发下会降级成只读/不可执行 shell。
- Codex StdIO 默认参数改为 `exec --json --sandbox danger-full-access --skip-git-repo-check -C . -`，让 AgentHub 选择的 workspace 同时作为 spawn cwd 和 Codex `-C` 工作根。
- 已用真实 Codex CLI 在 `C:\Users\Admin\Desktop\测试` 冒烟验证：可执行 PowerShell，并可定位 `D:\AgentHub开发版本`。

## 0.5.2 — ACP client fs handler（2026-06-16，`v0.5.2`）

**主题**：让 ACP agent 通过标准 client fs 方法访问 AgentHub 工作区。

- `initialize` 声明 `clientCapabilities.fs.readTextFile/writeTextFile=true`，接入 `fs/read_text_file` / `fs/write_text_file`。
- 每个 ACP session 绑定工作区根目录；绝对/相对路径都必须解析到 workspace 内，拒绝路径逃逸和符号链接逃逸。
- 直接 `fs/write_text_file` 复用 0.4.0 的 `write` 审批策略，避免绕过 `session/request_permission`。
- 本机已卸载旧版 `AgentHub 0.1.0`，安装并启动 `AgentHub 0.5.2`。

## 0.5.1 — ACP 权限请求接入审批门禁（2026-06-16，`v0.5.1`）

**主题**：把 ACP `session/request_permission` 接到 AgentHub 写/执行审批策略。

- ACP 权限请求规范化为 `write` / `exec` / 只读未知三类。
- `allow / ask / deny` 复用 per-agent 审批策略和既有审批弹窗。
- 只读或未知请求保持兼容放行。

## 0.5.0 — ACP 统一接入（2026-06-16，`v0.5.0`）

**主题**：让支持 ACP（Agent Client Protocol）的本地 agent CLI 统一接入，结构化活动开箱即有。

- 运行时探查发现 hermes / openclaw / minimax-code(opencode) **都支持 ACP**（JSON-RPC over stdio 标准），故以「一个 ACP 客户端适配器」取代 0.3.0 设想的「为三者各写文本活动解析器」——结构化活动（工具 / 文件 / 思考 / 正文）开箱即有，不靠脆弱的文本逆向。
- 新增 `adapters/acp-client.ts`（JSON-RPC 客户端 + `mapAcpUpdate` 纯函数映射）、`adapters/acp-adapter.ts`（`protocol:'acp'` + `acpDefaults`）、dispatcher `sendToAgentAcp` 路径；`protocol:'acp'` 全链路 + 设置→路由 ACP 后端选项 + 能力矩阵展示。
- 端到端握手已用真实 `opencode acp` 验证（`initialize` + `session/new` 互通）；`session/prompt` 完整对话流走同一套 JSON-RPC，待联机验证。
- 测试 149 passed（28 files）。设计：[DESIGN-0.5.0-acp.md](./DESIGN-0.5.0-acp.md)。

## 0.4.0 — 审批门禁 + proxy 工具透传（2026-06-16，`v0.4.0`）

**主题**：补齐 0.3.0 列入后续的 Item K 两项。

- **写/执行审批门禁**（`agentic/approval.ts`）：per-agent × per-tool 的 `allow / ask / deny`（默认全 allow，零回归）；`ask` 运行时弹窗逐次审批，`deny` 直接挡下并回灌模型；只读工具永不门禁。配置 UI 在能力矩阵下方「审批策略」。
- **proxy Anthropic 入站工具透传**（`routing/proxy.ts`）：`/v1/messages` 解析入站 `tools`/`tool_choice`、保留 `tool_use`/`tool_result` 多轮结构、把上游 tool_calls 回写为 anthropic `tool_use` SSE 块。
- 测试 141 passed（27 files）。后续项见 [DESIGN-0.3.0-capability-parity.md](./DESIGN-0.3.0-capability-parity.md) §6。

## 0.3.0 — 全员 Agent 能力对齐（2026-06-16，`v0.3.0`）

**主题**：让所有接入 agent（openclaw/hermes/marvis/minimax-code）在能力上对齐 codex/claude，补齐审计缺口。

- HTTP 原生 agentic 回环**默认对所有 agent 开启**（`agentic/config.ts` v2：`mode='all'` + 显式停用名单 + v1 迁移）；未绑定工作区时只读（写/执行限工作区内，拒绝 `..`/绝对路径逃逸）。
- 工作区 `bootstrapFiles` 作为项目级上下文注入（`workspace.ts#bootstrapContext`，路径沙箱 + 字符上限）。
- 多行提示词保真、thinking 对齐 stdio 路径、技能可 UI 编辑、能力矩阵「默认全员 Agentic」总开关。
- 测试 126 passed（25 files）+ 5 个新测试。设计：[DESIGN-0.3.0-capability-parity.md](./DESIGN-0.3.0-capability-parity.md)。

## 0.2.4 — 修复 dev 渲染黑屏（2026-06-15，`v0.2.4`）

- `npm run dev` 渲染黑屏：`index.html` 的 CSP `script-src 'self'` 拦截了 `@vitejs/plugin-react` 在 dev 注入的 inline Fast Refresh 脚本，导致 React 无法挂载。新增仅开发（`apply:'serve'`）生效的 `dev-csp-relax` 插件放行 `'unsafe-inline'/'unsafe-eval'`；生产构建 CSP 保持严格，安全性不受影响。
